about-me에는 자신에 대한 설명을 rules에는 프로젝트가 알아야할 규칙들을 정의해주는 공간입니다.

conventions에는 코딩 컨벤션이나 팀내 컨벤션을 적어주시면 됩니다.

code-writer와 test-code-writer는 각각 자신의 conventions md 파일을 참고하여 진행하므로 컨벤션을 알맞게 수정해서 사용하세요.

project_structure에는 PM이 기획을 진행할 때, 접근해야할 프로젝트들의 경로와 설명을 적어주세요.

예시
```
## example-project

### 경로
/user/project/example-project

### 설명
예시 프로젝트로 모든 기능을 포함하고 있는 프로젝트
```
